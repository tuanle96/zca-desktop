# zca-desktop v0.1.1

Production cloud endpoint and Resend magic-link email release.

## Download

- GitHub asset: `Zalo.Desktop_0.1.1_universal.dmg`
- Local build artifact: `Zalo Desktop_0.1.1_universal.dmg`
- SHA-256: `19e515806182f1e1e9dd8c631c6fc14474dd8574645cb3d047b685981b9aca16`
- macOS architecture: universal `x86_64 arm64`
- Bundle identifier: `app.zca.desktop`

## Highlights

- Desktop and mobile cloud auth now default to `https://zca.tuanle.dev` instead of localhost.
- Cloud server magic-link delivery now supports Resend via `ZCA_CLOUD_RESEND_API_KEY`, before SMTP/webhook fallback.
- Production deployment examples document the Resend sender `ZCA Cloud <no-reply@zca.tuanle.dev>`.
- macOS DMG is Developer ID signed, notarized, stapled, and Gatekeeper accepted.

## Verification

- `POST https://zca.tuanle.dev/api/v1/auth/magic-link/request` returned `{"sent":true,"expiresInSecs":600,"devMagicToken":null}`.
- `bun --cwd apps/desktop tauri build --target universal-apple-darwin --bundles dmg --ci` passed.
- Apple notarization submission `f36aef68-187e-49ea-a232-b576a661dacb` returned `Accepted`.
- `xcrun stapler staple` and `xcrun stapler validate` passed for the DMG.
- `codesign --verify --verbose=4` passed for the DMG.
- Mounted app `codesign --verify --deep --strict --verbose=4` passed.
- `spctl -a -vv -t install` accepted the DMG as `Notarized Developer ID`.
- `spctl -a -vv` accepted the mounted app as `Notarized Developer ID`.

## Caveats

- Harness readiness is currently blocked by the advisor-proof artifact issue in the local agent-harness runtime, not by the macOS release artifact.
- Unofficial client, not affiliated with Zalo or VNG.
- Zalo credentials are bearer tokens; do not share imported credentials or debug captures.
- Windows and Linux signed installers are not included in this release.
