# zca-desktop v0.1.7

Release date: 2026-06-08

## Install

Download the universal macOS DMG from the GitHub release and open it:

```sh
open Zalo.Desktop_0.1.7_universal.dmg
```

Or install through Homebrew Cask:

```sh
brew tap tuanle96/tap
brew update --force --quiet
brew reinstall --cask tuanle96/tap/zca-desktop
```

The DMG is Developer ID signed, notarized, and stapled, so it opens normally on
macOS without a Gatekeeper workaround.

## Highlights

- Google/GitHub browser login for cloud device sessions.
- Provider-owned OAuth tokens stay on the cloud backend; the desktop receives
  only a one-time ZCA device-code exchange result.
- Desktop stores cloud device tokens in Keychain and keeps email magic-link
  login as a fallback.
- The desktop preflights `GET /api/v1/auth/oauth/providers` so it can report
  unconfigured Google/GitHub OAuth before opening the browser.
- Message reaction picker with the full reaction set, plus a tidier inline image
  attachment preview.
- Cloud realtime now sends an immediate SSE `connected` event plus heartbeat so
  Cloudflare does not time out idle streams before the first synced message.
- Deployment docs now include Google/GitHub callback URLs and provider
  configuration checks.

## Server OAuth setup

Configure these backend environment variables to enable browser device login:

```sh
ZCA_CLOUD_GOOGLE_CLIENT_ID=...
ZCA_CLOUD_GOOGLE_CLIENT_SECRET=...
ZCA_CLOUD_GITHUB_CLIENT_ID=...
ZCA_CLOUD_GITHUB_CLIENT_SECRET=...
```

Provider callback URLs:

- `https://zca.tuanle.dev/auth/oauth/google/callback`
- `https://zca.tuanle.dev/auth/oauth/github/callback`

## Smoke Results

Interactive browser-login roundtrip ran against `https://zca.tuanle.dev` on
2026-06-08 using a freshly built and installed `Zalo Desktop.app` (loopback
callback listener on `127.0.0.1:37886`, single instance):

- **GitHub** — `oauth-callback: device linked via oauth code`; server logged
  `oauth_login_completed` then `oauth_device_registered`; the one-time desktop
  code was consumed and a device token was issued; realtime connected.
- **Google** — same chain confirmed; a new `oauth_user_identities` row was
  upserted and the device session connected.

Each run was cross-checked against the desktop log, the `audit_events` table,
and the `oauth_desktop_codes` / `oauth_user_identities` rows.
