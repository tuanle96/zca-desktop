Cloud Attachment Send

Status: pass

Backend now sends uploaded encrypted cloud files through hosted Zalo sessions and persists refresh-safe outgoing file metadata for cloud-only desktop.

Implemented

  • POST /api/v1/accounts/:account_id/send/file validates device auth, account ownership, and file account ownership.
  • Encrypted object-store blobs are decrypted only in backend memory, verified against stored size/hash, then passed to zca-rust as AttachmentSource::Data.
  • Outgoing file messages are stored with encrypted rich metadata, so refresh keeps the attachment bubble.
  • Tauri and Svelte cloud upload flow now call cloud_send_file after encrypted blob upload.
